Problemas Críticos Identificados

Bloqueadores que impedem evolução e escala

⏱️

Timeouts Crônicos SQL

Queries SQL sobre 63M contatos levam 45-60 min (timeout). Módulo Market One to One (70% do valor do contrato MC) não executa. Erro visível para dealers.

🔄

DW DataCare — Hub Obrigatório

TODAS as 10+ fontes passam pelo DW antes de qualquer consumo. Batch 24h. VUC presa dentro do DW — não acessível para ativação direta. Core recebe tudo indiscriminadamente.

⚠️

Limite MC Connect Excedido (195%)

350+ objetos sincronizados vs limite documentado de 180. Campanhas param quando novos objetos são adicionados. Jornadas com dados novos (telemetria, formulários) ficam bloqueadas.

👥

Fragmentação de Identidade

CPF não resolve tudo. Ownership não funciona quando cliente troca de dealer. VUC duplicada: lógica no DW + Identity Resolution parcial no DC. Sistema duplica histórico em vez de unificar.

Roadmap Executivo

6 Ondas de Migração • Descomissionamento Gradual do DW • Critérios de Saída por Onda

ONDA 1

Discovery

Critério de Saída
Objetivo: Mapear estado atual e planejar migração

• Inventariar 100% regras DW (transformações, deduplicação, VUC)
• Validar Data Lake Databricks (já em produção)
• Mapear 10+ fontes (DMS, HSF, dealers 2w/4w)
• Dimensionar consumo créditos Data360
• Identificar 350+ objetos MC Connect
• Documentar protocolos legados (XML-SOAP, WSO2, JSON-REST)
ONDA 2

Quick Win

Critério de Saída
Objetivo: Resolver bloqueador P0 de timeouts

• Piloto: 5-10 dealers migrados para Data360
• Segmentação reduzida de 45-60 min para < 5 min
• Módulo Market One to One (70% valor contrato) operando
• Dealers conseguem segmentar sem erro de timeout
• Validar consumo real de créditos Data360
ONDA 3

Identity Resolution

Critério de Saída
Objetivo: Unificar 63M contatos fragmentados

• Migrar lógica VUC do DW para Data360 Unified Profile
• Configurar rulesets (CPF, email+nome, phone+nome, VIN+dealer)
• Resolver ownership quando cliente troca de dealer
• Match rate validado (baseline vs. Data360)
• Base deduplicada operando
ONDA 4

Ativação Direta

Critério de Saída
Objetivo: Ativar novos canais sem passar pelo Core

• WhatsApp conversacional operando (Agent Force SDR)
• Push Notification mobile ativo
• Marketing Cloud ativando direto do Data360
• Data Lake (Databricks) recebendo histórico/analytics
• Writeback seletivo para Core (Oportunidades, Cases, Accounts)
ONDA 5

Rollout Dealers

Critério de Saída
Objetivo: Escalar piloto para totalidade de dealers

• Escalar de 5-10 dealers piloto → 1.000 dealers
• Todos dealers usando segmentação no Data360
• Sistema estável com 63M contatos ativos
• 392 jornadas Marketing Cloud operando
• Consumo créditos dentro do previsto
ONDA 6

Descomissionamento

Critério de Saída
Objetivo: Desligar DW e finalizar migração

• DW desligado permanentemente
• DMS conectando direto ao Data360 e Data Lake
• MuleSoft orquestrando sem DW intermediário
• Core leve: 350+ objetos → ~50 objetos
• Migração 100% completa, DW offline sem impacto

Benefícios Mensuráveis

Impactos quantificáveis da transformação

~95%
Latência Reduzida
Antes: Batch de 24h do DW. Dados de ontem disponíveis apenas amanhã.

Depois: Real-time via Data360 Streams. Dados disponíveis minutos após ingestão. Ativação imediata de campanhas sem espera por processamento noturno. Dealers veem dados atualizados o tempo todo.
< 5 min
Segmentação Rápida
Antes: Queries SQL sobre 63M contatos levam 45-60 min e causam timeout. Módulo Market One to One (70% do valor do contrato MC) não executa.

Depois: Segmentação drag & drop no Data360, pré-calculada em tempo de ingestão. Resultado em < 5 min. Dealers segmentam sem erros.
350+ → 50
MC Connect Leve
Antes: 350+ objetos sincronizados via MC Connect (limite documentado: 180). Campanhas não sincronizam quando novos objetos são adicionados.

Depois: Writeback seletivo. Core recebe apenas Oportunidades, Cases e Accounts (~50 objetos). Marketing opera independente consumindo direto do Data360.
63M
Base Unificada
Antes: Mesmo cliente com múltiplos IDs (app, site, concessionária). CPF como chave mas não resolve tudo. Ownership quebra quando cliente troca de dealer.

Depois: Identity Resolution nativa do Data360. Rulesets específicos (CPF, email+nome, phone+nome, VIN+dealer). Base deduplicada. Comunicações únicas por cliente real.
100%
Timeouts Eliminados
Antes: Queries SQL pesadas sobre 63M contatos causam timeout crônico. Dealers veem erro na interface. Campanhas não executam.

Depois: Segmentação pré-materializada no Data360. Sem queries SQL na hora de ativar. Dealers nunca mais veem erro de timeout. Módulo campaigns 100% operacional.
4 canais
Novos Canais
Canais adicionados:
• WhatsApp conversacional (Agent Force SDR já em produção desde 05/2026)
• Push Notification mobile nativo
• Data Lake Databricks (analytics + histórico)
• Agent Force integrado

Todos ativando direto do Data360 sem passar pelo Core.
